# M0 — Kernel & Fundação — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (inline) ou superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Construir o `packages/spec` (Zod) como única fonte de verdade do spec JSON — 14 primitivos + tipos complexos + eventos/ações + `specVersion` — com fixtures golden e testes, sobre a fundação de um monorepo pnpm.

**Architecture:** Monorepo pnpm. O kernel é um pacote TS puro (`@widgetmill/spec`) sem dependências internas, exportando schemas Zod compostos por união discriminada em `type`, com recursão via `z.lazy`. Tipos TS derivam por `z.infer`. Fixtures `.json` canônicos são validados nos testes (e, no M1, parseados pelo Dart — trava o drift).

**Tech Stack:** pnpm 10, TypeScript 5, Zod 3, Vitest, ESLint + Prettier. (Node 18.18+ p/ o kernel; Node 20 recomendado nos apps.)

**Princípios:** DRY (binding/cor/enum num só lugar), YAGNI (sem codegen, sem descriptors ainda — isso é M2), TDD (teste vermelho → mínimo verde → commit), arquivos pequenos de responsabilidade única (1 schema por arquivo), commits frequentes.

## Estrutura de arquivos (decomposição)

```
packages/spec/
├─ package.json            # @widgetmill/spec, type:module, scripts test/typecheck/lint
├─ tsconfig.json
├─ vitest.config.ts
├─ src/
│  ├─ bindable.ts          # Bindable(inner) + regex BINDING/TOKEN
│  ├─ scalars.ts           # Color, Hex; helpers de enum
│  ├─ complex/
│  │  ├─ edgeInsets.ts · textStyle.ts · boxDecoration.ts
│  │  ├─ borderRadius.ts · alignment.ts · constraints.ts
│  │  └─ index.ts
│  ├─ actions/
│  │  └─ index.ts          # Action (union por type) + Events
│  ├─ nodes/
│  │  ├─ container.ts · column.ts · row.ts · stack.ts · positioned.ts
│  │  ├─ text.ts · image.ts · icon.ts · button.ts
│  │  ├─ sizedBox.ts · padding.ts · center.ts
│  │  ├─ expanded.ts · flexible.ts · spacer.ts · gestureDetector.ts
│  │  └─ index.ts          # lista de nós para a união
│  ├─ tree.ts              # Node = z.lazy(union); helpers child()/children()
│  ├─ schema.ts            # PropDef, WidgetSpec (specVersion, slug, name, kind, propsSchema, tree)
│  └─ index.ts             # superfície pública
├─ fixtures/
│  ├─ hello-card.json      # container>column>text (fatia verde)
│  ├─ text-only.json
│  ├─ column-spacing.json
│  └─ primary_button.json  # exemplo da spec §6
└─ test/
   └─ fixtures.test.ts     # valida todos os fixtures + casos inválidos
```

**Recursão sem import circular:** os nós importam `child`/`children` de `tree.ts`, que expõem `z.lazy(() => Node)`. `Node` é a união discriminada montada em `tree.ts` a partir de `nodes/index.ts`. Como `z.lazy` só avalia o thunk em `.parse()`, os bindings ESM já estão resolvidos — padrão recursivo idiomático do Zod.

## Task T0.3: Binding e escalares (TDD)

**Files:** `src/bindable.ts`, `src/scalars.ts`, `src/bindable.test.ts`

- [ ] **Step 1: teste vermelho** — `Bindable(z.number())` aceita `12`, `"{{x}}"`, `"$sm"`; rejeita `"abc"`. `Color` aceita `#FF1565C0`/`#1565C0`; rejeita `red`.
- [ ] **Step 2:** rodar `vitest run src/bindable.test.ts` → FAIL.
- [ ] **Step 3: implementação mínima**

```ts
// src/bindable.ts
import { z } from "zod";
export const BINDING = /^\{\{\s*[\w.]+\s*\}\}$/;
export const TOKEN = /^\$[\w.]+$/;
export const Bindable = <T extends z.ZodTypeAny>(inner: T) =>
  z.union([inner, z.string().regex(BINDING), z.string().regex(TOKEN)]);
```
```ts
// src/scalars.ts
import { z } from "zod";
export const Color = z.string().regex(/^#([0-9a-fA-F]{6}|[0-9a-fA-F]{8})$/);
```

- [ ] **Step 4:** rodar → PASS. **Step 5:** commit `feat(spec): bindable helper and color scalar`.

## Task T0.5: Nós do núcleo + união recursiva (TDD)

**Files:** `src/nodes/{text,column,container}.ts`, `src/nodes/index.ts`, `src/tree.ts`, `src/tree.test.ts`

- [ ] **Step 1: teste vermelho** — `Node.parse` aceita um `container { child: column { children: [text] } }`; rejeita `{ type: "naoexiste" }`; um `text` com `overflow:"banana"` é rejeitado.
- [ ] **Step 2:** rodar → FAIL.
- [ ] **Step 3: implementação** — `tree.ts` define a recursão:

```ts
// src/tree.ts
import { z } from "zod";
import { nodeOptions } from "./nodes/index.js";
export interface SpecNode {
  type: string; props?: Record<string, unknown>;
  events?: unknown; child?: SpecNode; children?: SpecNode[];
}
export const Node: z.ZodType<SpecNode> =
  z.lazy(() => z.discriminatedUnion("type", nodeOptions)) as unknown as z.ZodType<SpecNode>;
export const child = () => z.lazy(() => Node);
export const children = () => z.array(z.lazy(() => Node));
```
Cada nó (ex. `container.ts`) importa `child`/`children` de `../tree.js`; `text.ts` segue a elaboração §2.1. `nodes/index.ts` exporta `nodeOptions = [ContainerNode, ColumnNode, TextNode]` (cresce no T0.8).

- [ ] **Step 4:** rodar → PASS. **Step 5:** commit `feat(spec): core nodes (text/column/container) + recursive Node union`.

- [ ] **Step 3: implementação**

```ts
// src/schema.ts
import { z } from "zod";
import { Node } from "./tree.js";
export const PropDef = z.object({
  key: z.string(),
  type: z.enum(["string","int","double","bool","color","enum","edgeInsets","textStyle","boxDecoration"]),
  required: z.boolean().default(false),
  default: z.unknown().optional(),
});
export const WidgetSpec = z.object({
  specVersion: z.literal(1),
  slug: z.string(), name: z.string(),
  kind: z.enum(["primitive","composite"]),
  version: z.number().int().optional(),
  propsSchema: z.array(PropDef).default([]),
  tree: Node,
});
export type WidgetSpec = z.infer<typeof WidgetSpec>;
```
`src/index.ts` reexporta `Node`, `WidgetSpec`, `PropDef`, complexos e helpers.
